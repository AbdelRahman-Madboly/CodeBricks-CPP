1. **Read the problem** ‚Äî Understand inputs, outputs, constraints
2. **Think before coding** ‚Äî Plan your approach on paper
3. **Write solution** ‚Äî Implement step by step
4. **Test thoroughly** ‚Äî Edge cases, large inputs
5. **Optimize** ‚Äî Can you improve time/space complexity?
6. **Review** ‚Äî Compare with reference solutions
